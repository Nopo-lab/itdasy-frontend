# T-500 plan — 앱 심사 전 신뢰 회복 1차

## 판단

이번 작업은 노란불이다. 스테이징 프론트와 제출 문서만 고친다. 운영 레포, 운영 DB, 결제, 네이티브 앱 설정은 건드리지 않는다.

## 실행 순서

1. 티켓과 전략 문서 작성.
2. 로그인 화면 잠금 상태를 정리해 첫 화면 뒤쪽 앱 조각을 숨김.
3. iPhone 앱에서는 Google/카카오 로그인 버튼을 숨김.
4. 내 샵의 플랜 문구를 실제 플랜 값과 맞춰 보이게 조정.
5. 개인정보/삭제/심사 문서의 연락처와 Apple 로그인 문구 수정.
6. 자동 확인과 브라우저 화면 확인.
7. `.ai/SESSION_STATE.md`, `.ai/BOARD.md`, `.ai/FOR_USER.md` 갱신.

## 위험

| 위험 | 막는 방법 |
|---|---|
| iOS에서 소셜 로그인 버튼만 빠져 가입 길이 줄어듦 | 이메일 로그인은 그대로 둔다. 웹과 Android는 기존 버튼을 유지한다. |
| 플랜 표시가 늦게 바뀜 | 처음에는 Free로 보이고, 서버에서 상태를 받아오면 실제 플랜으로 바뀌게 한다. |
| 문서가 제출 전 상태와 달라짐 | Apple 로그인은 "구현됨"이라고 쓰지 않고, 현재 iOS 심사 빌드 기준으로 설명한다. |

## 남겨야 할 백엔드 일

- 심사용 데모 계정에 샘플 데이터 넣기.
- 서버에서 실패하는 요청 3개 확인.
- `/shop/settings` 저장 권한 확인.
